Gridly is the project that wasn't supposed to be one. I extracted the constraint-based layout engine from [Peakfolio](/projects/peakfolio/), wrote a README at 2am, and woke up to strangers filing issues.

## Accidental infrastructure

Two years later it's the piece of my work most people have touched without knowing my name — which is exactly what good infrastructure should be. Nobody thinks about the layout engine when the layout works.

## What maintaining it taught me

More about API design than any product role: every abstraction you expose is a promise you keep for years. Deprecations are conversations, breaking changes are debts, and the best feature request is often the one you decline.
