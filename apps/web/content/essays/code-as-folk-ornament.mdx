Folk ornament was never decoration for its own sake. The borders stitched into a tablecloth, the geometry carved into a roof beam — they were pattern languages: small vocabularies of marks, repeated and recombined, carrying identity from one generation of makers to the next.

The web has its own marks. Brackets, slashes, carets, the ASCII furniture of every codebase — we look at them all day and see syntax. But arrange them in a grid, let them cycle and repeat, and they behave exactly like embroidery: rhythmic, geometric, modular, unmistakably of their material.

## Repetition is the point

Ornament works because it repeats. A single cross-stitch is nothing; a border of them is a signature. The same is true of a loop that lays down glyphs on a grid — the individual character is arbitrary, the rhythm is the identity.

This is why generative systems feel so at home in brand identity work. A seeded random function is a weaving loom: same threads, same rules, endlessly varied cloth. No two covers ship identical, and yet every one is recognizably from the same hand.

## Constraints as inheritance

Folk patterns survived because their rules were simple enough to teach: these motifs, these colors, this grid. A design system is the same act of inheritance — the palette stays tight, the glyph set stays small, and whoever builds the next page inherits the fire, not the ashes.

Write your loops the way weavers counted threads. The output is ornament; the code is the craft.
